# Outdoor Risk Matrix

| Risk | Cause | Detection | Immediate Action | Recovery | Code Module | Test Method | Limitations |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Runaway command flood | `/cmd_vel` spike, stuck key, reconnect replay, NaN/inf, motor swap error, missing accel limit | command delta, type/finite checks, stale timestamp, process lock, IMU/yaw spike | clamp, accel limit, soft stop, emergency stop | manual reset, inspect serial ownership | `runaway_guard`, `command_sanitizer`, `acceleration_limiter`, `command_mux`, `estop` | unit tests, fake serial, spike injection | Without wheel encoders this detects command risk more than actual speed. |
| Dynamic obstacle | person, child, bike, car, pet, ball, leaves, gate opening | LiDAR front sector, TTC trend, camera optional | stop or speed limit | wait, replan, skip, return home | `collision_guard`, `ttc_guard`, `dynamic_obstacle_tracker`, `avoidance_path_generator` | box/person approach tests | Classification is limited without trained camera detector. |
| Avoidance failure | both sides blocked, narrow passage, no valid path, costmap stale, oscillation | local sectors unsafe, Nav2 failure, repeated recovery | stop | wait, cancel goal, replan, backup if rear clear, skip waypoint | `path_validator`, `recovery_behaviors`, `avoidance_path_generator` | blocked waypoint test | Open-loop base cannot guarantee precise recovery motion. |
| Localization failure | AMCL covariance, wrong initial pose, kidnapped robot, map mismatch, stale tf/odom, weak GPS | covariance, pose age, tf age | cancel goal and stop | relocalize, reset initial pose, manual drive | `localization_guard`, `covariance_monitor`, `relocalization` | covariance injection, tf stale test | GPS patrol disabled without GPS/RTK. |
| Mapping/map issue | dynamic objects in map, rain/snow/grass noise, save failure, old map | map quality score, scan mismatch, dynamic flag | stop patrol start, warn | remap, dynamic filter, validate map | `dynamic_filter`, `map_quality`, `map_io` | map mismatch test | Outdoor maps change; manual validation required. |
| LiDAR issue | stale scan, too few points, glass, sunlight, rain, dust, leaves, near dead zone | scan age, valid point count, range filters | stop or degraded mode | clean sensor, restart driver | `lidar_anomaly_filter`, `sensor_stale_guard`, `collision_guard` | unplug/stale scan test | LiDAR may miss glass and low objects. |
| Camera issue | backlight, night, shake, dirty lens, rain, frame drop | frame age, optional detector health | degraded mode; LiDAR-only safety | clean lens, improve lighting | `camera_health`, `object_detector` | frame drop simulation | Camera is optional; no camera means limited object classification. |
| Terrain issue | mud, sand, wet grass, slope, curb, pothole, drain, puddle | traversability label, IMU pitch/roll, manual observation | slow or stop | manual intervention, speed zone | `traversability_estimator`, `keepout_speed_zone_manager` | terrain low-speed trials | No encoder means slip/stuck confidence is limited. |
| Slip/stuck | command forward but pose not changing, IMU vibration, yaw anomaly | pose delta vs command, IMU trend | stop, backup if safe | recovery or manual rescue | `recovery_behaviors`, future `stuck_detector` | simulated pose freeze | Without encoders this is approximate. |
| System fault | Jetson heat, battery drop, disk full, log failure, container/node death, serial conflict | thermal, voltage, watchdog, lsof | stop, return home on low battery | restart node/container, cool down | `thermal_guard`, `battery_guard`, `process_watchdog`, `port_conflict_detector` | kill node, thermal/battery simulation | Battery voltage source must be calibrated. |
| Communication fault | serial disconnect, USB cable, Wi-Fi, stale ROS graph, action timeout | serial exceptions, topic age, action timeout | repeated stop, stop on disconnect | reconnect only after stop | `serial_json_client`, `launch_manager`, `sensor_stale_guard` | fake serial disconnect | Stop cannot be delivered if the link is physically broken. |
| Human intervention | manual override, E-Stop, pause/resume | keyboard/manual source priority | manual wins, clear queues, stop | reset before patrol resume | `estop`, `command_mux`, `patrol_state_machine` | manual override test | Requires trained E-Stop operator. |
